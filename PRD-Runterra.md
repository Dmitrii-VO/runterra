# PRD: Runterra — Платформа локальных беговых сообществ

**Версия:** 1.0
**Дата:** 2026-03-06
**Статус:** MVP

---

## 1. Краткое описание

Runterra — мобильное приложение (Android), которое объединяет локальные беговые клубы через геймификацию: клубы захватывают и удерживают территории города, регулярно проводя совместные пробежки с GPS-подтверждением. Приложение решает проблему разрозненности беговых сообществ и низкой мотивации к регулярным тренировкам.

## 3. Контекст и предпосылки

### Что происходит сейчас

Беговые клубы и тренеры организуют активности хаотично: через Telegram-чаты, Instagram, VK, сторонние сервисы. Нет единой точки входа, нет карты событий, нет общего рейтинга.

Существующие приложения для бега (Strava, Nike Run Club, Garmin) сфокусированы на **личных метриках** — темпе, пульсе, дистанции. Они не работают с локальными сообществами и не создают командной мотивации.

### Почему сейчас

- Беговое движение в России растёт: количество участников массовых стартов увеличивается каждый год.
- В Санкт-Петербурге сформировалось зрелое беговое сообщество с десятками активных клубов — идеальная среда для пилота.
- Геймификация в fitness-приложениях доказала свою эффективность (Zombies, Run!; Habitica; Strava Segments).
- Технологии GPS-валидации и картографические SDK доступны и зрелы.

### Что изменилось

Пользователи устали от индивидуальной гонки за личными рекордами. Растёт запрос на **социальный бег** — бегать вместе, с целью, с командой.

## 4. Цели и метрики успеха

### Стратегическая цель

Стать основной платформой для локальных беговых сообществ России, начав с Санкт-Петербурга.

### Цели MVP (горизонт: 3 месяца после запуска пилота)

| Цель | Ключевой результат | Целевое значение |
|------|-------------------|-----------------|
| Вовлечённость пользователей | WAU / MAU | >= 30% |
| Активность клубов | Среднее кол-во тренировок/клуб/неделю | >= 3 |
| Удержание | Возврат пользователей через 7 дней | >= 40% |
| Геймплей | % территорий с активным владельцем | >= 70% |
| Удовлетворённость | NPS среди лидеров клубов | >= 40 |
| Органический рост | Новые клубы без прямого привлечения | > 0 в месяц |

### Цели запуска (Фаза 1, первый месяц)

- Онбордить 10-15 беговых клубов-партнёров в СПб
- Провести первые 50 тренировок с GPS-чекинами
- Захватить первые 20 территорий

## 5. Целевые сегменты

### Первичный сегмент — Лидеры и организаторы беговых клубов

**Кто это:** основатели клубов, главные тренеры, организаторы регулярных пробежек.

**Их работа (Jobs to be Done):**
- Удерживать участников клуба активными между сезонными стартами
- Привлекать новых участников без больших затрат на маркетинг
- Иметь инструмент для планирования и учёта тренировок

**Почему выбираем их первыми:** они — мультипликаторы. Один лидер клуба приводит 10-50 участников.

---

### Вторичный сегмент — Социально-активные бегуны

**Кто это:** люди, которые регулярно бегают и ищут компанию или клуб. Возраст 20-40 лет, Android-устройства, городские жители.

**Их Jobs to be Done:**
- Найти людей для совместных пробежек рядом с домом
- Иметь внешнюю мотивацию выходить на пробежки регулярно
- Чувствовать принадлежность к сообществу

---

### Третичный сегмент — Меркатели (независимые бегуны)

**Кто это:** опытные бегуны без привязки к одному клубу. Хотят разнообразия маршрутов и знакомств.

**Ограничения:** за одну неделю могут помочь не более 3 разным клубам (защита от эксплуатации системы).

---

### Вне скопа MVP

- Организаторы коммерческих забегов (post-MVP)
- Бренды и партнёры (post-MVP)
- iOS-пользователи (post-MVP)

## 6. Ценностное предложение

### Для лидеров клубов

| Работа пользователя | Что даёт Runterra |
|---------------------|-------------------|
| Удержать участников активными | Система захвата территорий создаёт командную цель вне соревновательных сезонов |
| Привлечь новых участников | Карта клубов и событий — органический канал открытия |
| Планировать тренировки | Создание тренировок с геометкой, расписанием, уровнем сложности |
| Понимать активность клуба | Метрики: участники, территории, рейтинг в городе |

### Для бегунов

| Боль | Решение |
|------|---------|
| "Скучно бегать одному, нет мотивации" | Командное соревнование за территории — бег становится игрой |
| "Не знаю, где и когда тренируются клубы рядом" | Интерактивная карта событий и клубов |
| "Потерял ритм, не могу заставить себя выйти" | Push-уведомления от клуба, баллы за участие |

### Отличие от конкурентов

- **Strava:** фокус на личных рекордах, нет коммунити-геймификации
- **Nike Run Club:** нет карты локальных клубов, нет командного геймплея
- **Telegram-чаты:** нет структуры, нет карты, нет аналитики

Runterra — единственный инструмент, где **командная активность на карте города** является основным геймплеем.

## 7. Решение

### 7.1 Пользовательские сценарии (ключевые flows)

**Онбординг (3 шага):**
1. Регистрация через email/телефон (OTP) → имя → город → согласие на геолокацию
2. Первый экран: список ближайших клубов + предложение присоединиться или создать свой
3. Карта с подсказкой: "Серые зоны нейтральны. Твой клуб может захватить их!"

**Участие в тренировке:**
1. Найти тренировку на карте или в профиле клуба
2. Записаться → получить push за 1 час до старта
3. На месте: нажать "Начать пробежку" → GPS-трекинг в фоне
4. После завершения: "+10 баллов. Вклад в захват территории 'Марсово поле'"

**Захват территории:**
- Условие: 5 пробежек в зоне + 3 уникальных участника клуба за 7 дней
- Результат: территория меняет цвет на цвет клуба, все участники получают push "+50 баллов"

---

### 7.2 Ключевые функции

#### Карта (основа продукта)
- Интерактивная карта города (Yandex MapKit)
- Отображение территорий по статусам: нейтральная (серая), захваченная (цвет клуба), оспариваемая (жёлтая)
- Отображение активных клубов и предстоящих тренировок
- Фильтры: по дате, уровню подготовки, клубу/тренеру

#### Территории
- Зоны соответствуют 18 административным районам Санкт-Петербурга (данные из OpenStreetMap)
- Захват считается в метрах GPS-трека, попадающих в полигон района (Ray Casting)
- Перед пробежкой пользователь выбирает, какому клубу засчитывается вклад
- Удержание: минимум 2 пробежки/неделю с 2+ участниками
- Оспариваемая: нет активности 3 дня
- Нейтральная: нет активности 7 дней
- Карточка территории: владелец, таймер, история последних 3 смен

#### Клубы
- Любой пользователь может создать клуб (требует 3 участника + модерацию)
- Профиль: название, описание, город, зоны активности, расписание
- Роли: лидер → модераторы → участники
- Чат клуба (текстовый, WebSocket)
- Метрики клуба: участники, территории, рейтинг в городе
- Жизненный цикл: active → inactive (30 дней без активности) → нейтральные территории (60 дней)

#### Тренировки
- Создание: геометка, дата/время, уровень сложности, видимость (public/private)
- Запись и GPS-чекин (радиус 200-500 м от старта, мин. 1 км в пределах территории)
- Валидация скорости: 3-20 км/ч
- No-show политика: 3 пропуска подряд → ограничение записи на 7 дней
- Автоотмена: никто не записался за 1 час до старта
- Привязка шаблона тренировки (workoutId) к событию — тренер назначает план на занятие

#### Профиль пользователя
- Имя, уровень, принадлежность к клубу, статус "Меркатель"
- Статистика: участия, баллы, общий километраж, вклад в территории

#### Инструменты тренера (Athletic Pro)
- Конструктор структурированных тренировок: блоки (разминка, основная часть, заминка), сегменты с целевой пульсовой зоной, RPE, типом отдыха (Stand / Walk / Jog)
- Покрытие маршрута: шоссе, грунт, стадион
- Привязка плана к событию — тренер назначает шаблон на занятие
- Голосовые подсказки при переходе к следующему сегменту (flutter_tts)
- После пробежки: участник оставляет RPE (1-10) и заметку тренеру

#### Геймификация
- Баллы: +10 за участие, +50 за захват новой территории, +5/неделю за удержание
- Командный рейтинг клубов в городе
- Роль "Меркатель": помогает разным клубам (макс. 3 клуба/неделю)
  - **MVP-ограничение:** пробежки меркателей не начисляют очки территориям и не делают зону "оспариваемой". Эта механика запланирована на post-MVP.

#### Push-уведомления
- Новая тренировка от моего клуба
- Территория под угрозой
- Успешный захват
- Напоминание за 1 час до тренировки

#### Антифрод
- Валидация GPS: скорость, телепортации, mock locations detection
- Система флагов: 1 нарушение → предупреждение, 2 → бан check-in 7 дней, 3 → постоянный бан
- Система репортов от пользователей

### 7.3 Технология

| Слой | Стек |
|------|------|
| Mobile | Flutter (Android-first), Yandex MapKit, GoRouter, Firebase Auth |
| Backend | Node.js + TypeScript, Express, PostgreSQL, WebSocket (ws), Zod |
| Auth | Firebase Authentication (email/phone OTP) |
| Карты | Yandex MapKit (mobile), Yandex Maps JS API (admin) |
| Real-time | WebSocket (чат), Push через FCM |
| Геоданные | PostgreSQL + PostGIS (ST_Within, ST_Distance) |
| Admin | Next.js 14 |
| Инфраструктура | Cloud.ru, GitHub Actions CI |

---

### 7.4 Допущения

- Беговые клубы в СПб готовы перейти на новый инструмент, если он даёт видимое конкурентное преимущество (карта территорий)
- GPS-валидация достаточна для MVP без сложного антифрода
- Первые клубы будут онбордированы вручную через личные контакты
- Монетизация не нужна в MVP — фокус на росте пользовательской базы

## 8. Релиз

### Фаза 1 — Пилот в СПб (текущая)

**Что входит:**
- Карта с территориями (центральные районы + ключевые парки)
- Клубы: создание, вступление, чат, роли
- Тренировки: создание, запись, GPS-чекин
- Профиль пользователя и базовая статистика
- Захват и удержание территорий (cron job)
- Push-уведомления
- Админка: управление территориями, модерация клубов и тренеров

**Приоритетные зоны запуска:**
- Центральный район (Марсово поле, Летний сад)
- Крестовский и Елагин острова
- Набережные Невы и каналов

### Фаза 2 — Расширение в СПб

- Верификация тренеров (бейдж "Верифицированный тренер")
- Система репортов от пользователей (читинг, спам, неадекватное поведение)
- Улучшенная аналитика в админке (DAU/WAU/MAU, retention, churn клубов)
- Wear OS: полноценный релиз — управление пробежкой с часов, ЧСС-датчик *(backend готов, ADR-0009 принят)*

### Фаза 3 — Монетизация и второй город

- Инструменты масштабирования по городам (партиционирование БД по cityId)
- **Freemium для клубов:** бесплатный тариф (до 50 участников, до 3 территорий) + Premium ~990 руб/мес (без ограничений, расширенная аналитика, приоритет на карте, брендирование)
- Спонсорские/брендированные территории (партнёрство с парками и спортивными брендами)
- API для организаторов коммерческих забегов

### Post-MVP: Коммуникации

- **Личные чаты 1:1** между пользователями
- **Тренер ↔ участники:** отдельный чат тренера с подопечными
- **Подчаты клуба (каналы):** "Общий", "Тренировки", тематические *(backend уже реализован)*
- **Комментарии к событиям:** обсуждение под карточкой тренировки

### Post-MVP: Социальные функции

- Лента активности: что сделали участники моего клуба (пробежки, захваты)
- Фото/видео в постах клуба
- QR-check-in как альтернатива GPS (для помещений и зон с плохим сигналом)

### Post-MVP: Геймификация (расширение)

- **Механика меркателей:** пробежки без клуба делают зону "оспариваемой" *(отложено с MVP, ADR зафиксирован)*
- Система достижений и бейджей (личные и командные)
- Сезоны и лиги: ежеквартальный сброс рейтинга, финальная таблица
- Межгородские соревнования (клубы из разных городов)

### Post-MVP: Тренерские инструменты (расширение)

- Статистика группы: посещаемость, динамика, прогресс участников
- Расписание на несколько недель вперёд (тренировочный план)
- Аналитика пробежек участников (темп, зоны, RPE в динамике)

### Post-MVP: Интеграции и платформы

- **iOS приложение** *(Flutter — перенос минимальный)*
- **Apple Watch** *(требует iOS-приложения)*
- **Strava:** импорт/экспорт пробежек
- **Garmin, Polar, Suunto:** синхронизация треков

### Не входит в roadmap

- NFT / токены
- Маркетплейс мерча
- Сложная прокачка персонажей / RPG-механики
