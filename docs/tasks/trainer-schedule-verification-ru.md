# Чек-лист: Этап 5 — Интерфейс Тренера (Mobile)

Используйте этот список для проверки корректности реализации функционала управления расписанием и составом клуба.

## 1. Доступ и интеграция (Экран деталей клуба)
- [x] **Кнопки управления:** Для пользователей с ролью «Лидер» или «Тренер» в деталях клуба виден блок «Управление тренера».
- [x] **Ограничение доступа:** Обычные участники (`member`) не видят кнопок управления расписанием и планами. (Проверено в коде: условие `club.userRole`)
- [x] **Навигация:** Кнопка «Расписание» открывает экран шаблонов, кнопка «Состав и планы» открывает список участников.

## 2. Управление недельным расписанием (Шаблоны)
- [x] **Селектор дней:** В верхней части экрана есть переключатель Пн–Вс (7 дней).
- [x] **Список заданий:** При смене дня отображаются соответствующие элементы шаблона. (Исправлено выравнивание `dayOfWeek`)
- [x] **Добавление заметки:** 
    - [x] Через кнопку `+` можно создать новую заметку (Note). (Исправлены названия кнопок в меню)
    - [x] Указывается время (HH:mm) и текст задания.
    - [x] Заметка сохраняется и появляется в списке выбранного дня. (Исправлена валидация API)
- [x] **Удаление:** Элемент удаляется из списка при нажатии на иконку удаления.
- [x] **Обработка пустого состояния:** Если заданий на день нет, отображается текст «Нет данных».

## 3. Состав клуба (Roster)
- [x] **Список участников:** Отображаются все активные члены клуба.
- [x] **Тип плана:** У каждого участника виден статус плана: **«Клубный»** или **«Личный»**.
- [x] **Цветовая индикация:** Статусы визуально отличаются (синий для клубного, фиолетовый для личного). (Исправлено: цвета стали контрастнее)
- [x] **Переход:** Тап по участнику открывает экран управления его персональным расписанием.

## 4. Индивидуальные планы (Personal Schedule)
- [x] **Просмотр текущего плана:** Экран показывает персональный шаблон участника.
- [x] **Замена плана:** Кнопка «Заменить персональный план» позволяет перезаписать текущий шаблон (в MVP — установка тестового задания). (Исправлена валидация API)
- [x] **Обновление статуса:** После назначения личного плана статус участника в общем списке (Roster) меняется автоматически. (Исправлено: добавлен принудительный релоад списка при возврате)

## 5. Техническая реализация и UX
- [x] **Локализация:** Все тексты соответствуют русскому языку. (Исправлено: переведены экраны Расписания, Состава и Плана)
- [ ] **Загрузка данных:** Присутствуют индикаторы загрузки при обращениях к API.
- [ ] **Обработка ошибок:** При сбоях сети выводятся соответствующие уведомления (SnackBar).
- [x] **Обновление по требованию:** На экранах расписания и состава работает кнопка «Обновить» в AppBar. (Исправлено: работает кнопка и RefreshIndicator)
