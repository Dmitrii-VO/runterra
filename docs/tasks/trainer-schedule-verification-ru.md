# Чек-лист: Этап 5 — Интерфейс Тренера (Mobile)

Используйте этот список для проверки корректности реализации функционала управления расписанием и составом клуба.

## 1. Доступ и интеграция (Экран деталей клуба)
- [ ] **Кнопки управления:** Для пользователей с ролью «Лидер» или «Тренер» в деталях клуба виден блок «Управление тренера».
- [ ] **Ограничение доступа:** Обычные участники (`member`) не видят кнопок управления расписанием и планами.
- [ ] **Навигация:** Кнопка «Расписание» открывает экран шаблонов, кнопка «Состав и планы» открывает список участников.

## 2. Управление недельным расписанием (Шаблоны)
- [ ] **Селектор дней:** В верхней части экрана есть переключатель Пн–Вс (7 дней).
- [ ] **Список заданий:** При смене дня отображаются соответствующие элементы шаблона.
- [ ] **Добавление заметки:** 
    - [ ] Через кнопку `+` можно создать новую заметку (Note).
    - [ ] Указывается время (HH:mm) и текст задания.
    - [ ] Заметка сохраняется и появляется в списке выбранного дня.
- [ ] **Удаление:** Элемент удаляется из списка при нажатии на иконку удаления.
- [ ] **Обработка пустого состояния:** Если заданий на день нет, отображается текст «Нет данных».

## 3. Состав клуба (Roster)
- [ ] **Список участников:** Отображаются все активные члены клуба.
- [ ] **Тип плана:** У каждого участника виден статус плана: **«Клубный»** или **«Личный»**.
- [ ] **Цветовая индикация:** Статусы визуально отличаются (синий для клубного, фиолетовый для личного).
- [ ] **Переход:** Тап по участнику открывает экран управления его персональным расписанием.

## 4. Индивидуальные планы (Personal Schedule)
- [ ] **Просмотр текущего плана:** Экран показывает персональный шаблон участника.
- [ ] **Замена плана:** Кнопка «Заменить персональный план» позволяет перезаписать текущий шаблон (в MVP — установка тестового задания).
- [ ] **Обновление статуса:** После назначения личного плана статус участника в общем списке (Roster) меняется автоматически.

## 5. Техническая реализация и UX
- [ ] **Локализация:** Все тексты соответствуют русскому языку.
- [ ] **Загрузка данных:** Присутствуют индикаторы загрузки при обращениях к API.
- [ ] **Обработка ошибок:** При сбоях сети выводятся соответствующие уведомления (SnackBar).
- [ ] **Обновление по требованию:** На экранах расписания и состава работает кнопка «Обновить» в AppBar.
