# 2026-02-28: Профессиональный конструктор тренировок (Athletic Pro)

## Что было сделано
В рамках задачи внедрения профессионального функционала для тренеров по легкой атлетике и улучшения обратной связи:
- **Data Architecture (Models):** В модели тренировок (`Workout`) добавлено поле `blocks` (JSONB массив блоков `WorkoutBlock`, содержащих `WorkoutSegment`), а также `surface` для выбора покрытия (Шоссе, Грунт, Стадион). На бекенде создана соответствующая миграция `032_workouts_pro_structure.sql`.
- **Локализация:** В файлы локализации `app_ru.arb` и `app_en.arb` добавлены новые ключи для элементов конструктора тренировок (Зона интенсивности, RPE, Тип отдыха, Покрытие и др.).
- **Mobile UI (Конструктор):** Значительно расширен `WorkoutFormScreen`. Добавлен UI-редактор блоков тренировок с возможностью указания времени, дистанции, типа сегмента, целевой пульсовой зоны, видео-инструкций и типа отдыха (Stand, Walk, Jog).
- **События (Events Integration):** В `EventsService` (mobile) и API (backend) добавлена возможность привязки `workoutId` при создании тренировки (`POST /api/events`), что позволяет тренеру назначать шаблон на определенное занятие. В `CreateEventScreen` добавлен соответствующий UI-селектор.
- **Пробежки (Run Tracking & Feedback):**
  - Экран `RunTrackingScreen` научился отслеживать выполнение структурированной тренировки.
  - Добавлена интеграция `flutter_tts` для голосовых подсказок при переходе к следующему сегменту.
  - При завершении пробежки теперь запрашивается **RPE (усилие 1-10)** и **заметки для тренера**.
  - Бекенд (`runs.routes.ts`, `RunsRepository`) обновлен и умеет сохранять новые поля через миграцию `033_runs_feedback.sql`.

## Результат
Приложение переведено на новую архитектуру структурированных тренировочных планов, способную удовлетворить потребности профессиональных тренеров и повысить качество собираемой статистики о тренировочном процессе. Все изменения успешно проходят проверку статического анализатора и тесты бекенда.
