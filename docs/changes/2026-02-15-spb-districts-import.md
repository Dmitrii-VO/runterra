# Импорт административных районов СПБ (Real Zones)

**Дата:** 2026-02-15
**Задача:** Заменить сгенерированную сетку квадратов на реальные административные районы Санкт-Петербурга (18 районов) согласно сайту `spb.ginfo.ru`.

## Реализовано

### Данные

- Импортированы границы 18 административных районов Санкт-Петербурга из OpenStreetMap (Overpass API).
- Использован `osmtogeojson` для конвертации и скрипт слияния для подготовки единого GeoJSON.
- Данные сохранены в `backend/scripts/data/final_districts.geojson`.

### Backend

- Создан генератор `backend/scripts/generate-spb-territories.js`, который преобразует GeoJSON в TypeScript-конфигурацию.
- Сгенерирован файл `backend/src/modules/territories/spb-districts.data.ts`, содержащий массив конфигураций территорий с реальной геометрией.
- Обновлен `backend/src/modules/territories/territories.config.ts`:
  - Убрана заглушка `SPB_TERRITORIES_CONFIG` с ручными квадратами.
  - Подключен импорт `SPB_DISTRICTS_DATA`.
  - Теперь территории соответствуют реальным районам (Адмиралтейский, Василеостровский и т.д.).

### Верификация

- Запущен скрипт `audit-territories.ts`.
- Результат: 18 территорий, ошибок геометрии нет, перекрытий нет (корректная топология).

## Следующие шаги

- В будущем, при переходе на БД, использовать этот же GeoJSON или скрипт миграции для заполнения таблицы `territories`.
- Проверить производительность на мобильном клиенте (некоторые полигоны могут быть детализированными). При необходимости добавить шаг упрощения (Douglas-Peucker) в скрипт генерации.
