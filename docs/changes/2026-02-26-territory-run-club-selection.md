# Выбор клуба для зачета и фикс багов карты

**Дата:** 2026-02-26

## Описание изменений

Реализованы задачи из `PLAN.md` (Выбор клуба для зачета и исправление бага карты):

1. **Модели данных (`mobile/lib/shared/models/run_session.dart`)**:
   - Добавлено опциональное поле `scoringClubId`.
   - Обновлен конструктор и добавлен метод `copyWith` для удобного обновления полей сессии.

2. **Сервис пробежек (`mobile/lib/shared/api/run_service.dart`)**:
   - Обновлен метод `startRun` для принятия `scoringClubId`.
   - Настроено сохранение `scoringClubId` в `_currentSession` при всех обновлениях метрик, паузе и возобновлении трекинга.
   - Метод `submitRun` теперь по умолчанию берет `scoringClubId` из `_currentSession`, если он не передан явно (для сохранения совместимости с фоллбэком).

3. **Экран трекинга (`mobile/lib/features/run/run_tracking_screen.dart`)**:
   - Добавлено локальное состояние списка клубов (`_myClubs`) и выбранного клуба (`_selectedScoringClub`).
   - Метод `_loadClubs()` загружает список клубов пользователя и выбирает текущий активный клуб по умолчанию (по `ServiceLocator.currentClubService.currentClubId`).
   - В UI перед началом пробежки добавлена карточка с названием выбранного клуба.
   - Добавлен диалог-шторка (`_showPreRunClubSelectionDialog`) для выбора или смены клуба перед стартом.
   - В метод `_startRun` передается ID выбранного клуба, который сохраняется в сессии и отправляется на бэкенд при завершении тренировки.

4. **Исправление карты (`mobile/lib/features/map/map_screen.dart`)**:
   - Введен флаг `_isSheetShowing` для предотвращения множественного открытия BottomSheet.
   - Защита применяется в методе `_showTerritoryBottomSheet`, блокируя вызовы, если шторка уже показана, и сбрасывая флаг через `whenComplete` после ее закрытия. Это устраняет баг, когда клик на границе районов открывал несколько окон подряд.
