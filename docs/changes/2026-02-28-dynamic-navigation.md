# Динамическая навигация (Скрытие вкладок) - 2026-02-28

## Описание
Реализована логика скрытия вкладок в нижнем меню приложения для пользователей, у которых нет активных клубов или тренеров. Это упрощает интерфейс для новых пользователей.

## Изменения

### Backend
- Добавлен метод `getNavigationStatus(userId)` в `UsersRepository`.
- Добавлен эндпоинт `GET /api/users/me/nav-status`.
- Логика проверки:
  - `hasClubs`: наличие в `club_members` со статусом `active`.
  - `hasTrainers`: наличие в группах тренера, личных контрактах или наличие роли тренера/лидера в любом клубе.

### Mobile
- Создана модель `UserNavStatus`.
- В `UsersService` добавлен метод `getNavigationStatus()`.
- Реализован `UserNavStatusNotifier` (синглтон) для управления состоянием видимости вкладок.
- Обновлен `BottomNav`:
  - Вкладка **"Сообщения"** скрыта, если `hasClubs == false` и `hasTrainers == false`.
  - Вкладка **"События"** скрыта, если `hasClubs == false`.
  - Реализован маппинг индексов между визуальным меню и ветками `GoRouter`.

## Влияние на UX
Новый пользователь при первом входе видит только 3 вкладки: **Карта**, **Пробежка**, **Профиль**. После вступления в клуб или найма тренера (через Карту или Профиль), соответствующие разделы автоматически появляются в меню.
