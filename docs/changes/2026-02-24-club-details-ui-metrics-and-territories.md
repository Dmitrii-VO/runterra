# Улучшения UI деталей клуба и логики территорий (2026-02-24)

## Контекст
Был получен фидбек по улучшению экрана деталей клуба (`ClubDetailsScreen`) и отображению прогресса территорий:
1. Очистить главный экран от длинных списков (основатель, участники).
2. Сделать метрики (Участники, Территории, Рейтинг в городе) кликабельными.
3. Отображать прогресс у каждого участника клуба.
4. При клике на "Территории" показывать не переход на карту, а список (Bottom Sheet) территорий с прогрессом захвата клуба.
5. Исправить отображение территорий без прогресса (скрыть их) и локализовать статусы.

## Выполненные изменения

### 1. Бэкенд
- **`club_members.repository.ts`:**
  - Обновлен метод `findMembersByClub`: добавлен SQL `LEFT JOIN runs` для подсчета суммарной дистанции (`total_distance`) по всем завершенным пробежкам участника в рамках клуба (`r.scoring_club_id = cm.club_id OR r.scoring_club_id IS NULL`). Учет пробежек с `NULL` позволяет корректно засчитывать старые пробежки, созданные до введения `scoring_club_id`.
  - Модель `ClubMemberDetailDto` расширена полем `totalDistance`.
- **`territories.routes.ts` / `map.routes.ts`:**
  - Изменен фильтр по `clubId`. Ранее возвращались только территории, где клуб был абсолютным владельцем. Теперь возвращаются все территории (включая `status: 'contested'`), где клуб имеет какой-либо прогресс (`myClubProgress != null`).
  - Районы без прогресса для выбранного клуба исключаются из финальной выдачи.

### 2. Мобильный клиент
- **`ClubMemberModel` & `TerritoryModel`:**
  - В `ClubMemberModel` добавлено поле `totalDistance`.
  - В `TerritoryModel` добавлено поле `myClubProgress`.
- **`ClubDetailsScreen`:**
  - Удалены блоки «Основатель» и развернутый список «Участники» из основного `SingleChildScrollView`.
  - Карточки метрик (Участники, Территории, Рейтинг) обернуты в `InkWell` (через метод `_buildMetricChip` с коллбеком `onTap`).
  - По клику на «Участники» вызывается `_showMembersSheet`, где отображается список с ролями и прогрессом (`X.Y км`). У лидера есть возможность сменить роль участника через `trailing` иконку.
  - По клику на «Территории» вызывается `_showTerritoriesSheet`, где отображаются захваченные и оспариваемые территории клуба с учетом прогресса (`totalKm` и `gapToLeader`). Список отсортирован: сначала захваченные (где клуб — лидер), затем оспариваемые по убыванию `totalKm`.
  - По клику на «Рейтинг в городе» вызывается `_showLeaderboardSheet` (пока отображается заглушка "В разработке").
- **Локализация (`AppLocalizations`):**
  - Добавлены/использованы ключи `territoryLeading`, `territoryCaptured`, `territoryFree`, `territoryContested`, `territoryLocked`, `territoryUnknown` для замены сырых английских статусов на удобочитаемые (например, "Оспаривается (Вы лидируете: 17.1 км)").
  - Добавлена локализация для города "spb" -> "Санкт-Петербург" на экране деталей территории (`TerritoryDetailsScreen`).

### 3. Исправление багов CI (Flutter 3.24.5)
- Возникла ошибка при использовании метода `.withValues(alpha: ...)`, который отсутствует во Flutter 3.24.5. Все подобные вызовы заменены на `.withOpacity(...)`.
- В `android/build.gradle` добавлен репозиторий `maven { url "https://maven.yandex.ru/" }` для разрешения зависимости `maps.mobile`.
- Устранены ошибки несовпадения ARB файлов и сгенерированных Dart-классов путем обновления ключа `filterAll`.
