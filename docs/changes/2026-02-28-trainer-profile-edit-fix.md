# Исправление редактирования профиля тренера

**Дата**: 2026-02-28
**Статус**: Выполнено

## Описание изменений
Исправлена проблема, при которой экран редактирования профиля тренера открывался пустым, даже если профиль уже существует. Также добавлена возможность перехода к редактированию прямо из публичного профиля тренера для его владельца.

### Mobile
1.  **ProfileScreen (Личный кабинет)**:
    *   Переменная состояния `_hasTrainerProfile` (bool) заменена на `_trainerProfile` (`TrainerProfile?`).
    *   Метод `_loadTrainerProfile` теперь сохраняет объект профиля.
    *   Удален лишний пункт «Редактировать профиль» из списка действий, так как теперь редактирование доступно через публичный профиль. Это упрощает интерфейс.
2.  **TrainerEditProfileScreen (Редактирование)**:
    *   Добавлено состояние загрузки `_loading`.
    *   В `initState`, если профиль не был передан через `extra` (например, при прямом переходе по ссылке), выполняется автоматическая загрузка профиля текущего пользователя через `ServiceLocator.trainerService.getMyProfile()`.
    *   Форма инициализируется данными сразу после загрузки.
    *   Режим сохранения (создание или обновление) теперь определяется по наличию загруженного профиля `_loadedProfile`.
3.  **TrainerProfileScreen (Публичный профиль)**:
    *   В `AppBar` добавлена кнопка «Редактировать» (иконка карандаша).
    *   Кнопка отображается только в том случае, если просматриваемый профиль принадлежит текущему пользователю (`isMe`).
    *   При нажатии текущий объект профиля передается на экран редактирования.

## Верификация
- `flutter analyze`: Ошибок в измененных файлах не обнаружено.
- Проверены сценарии:
    - Переход из ЛК (карточка тренера -> Редактировать): поля заполнены.
    - Переход из публичного профиля (иконка в AppBar): поля заполнены.
    - Прямой переход на `/trainer/edit`: профиль загружается автоматически.

## Влияние на систему
- Улучшен UX для тренеров.
- Устранена путаница между созданием и редактированием профиля.
- Повышена устойчивость навигации.
