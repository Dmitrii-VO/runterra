## Фаза 1 — Quick wins

### 1.1. Город: `spb` → «Санкт-Петербург»

**Файлы:** `mobile/lib/features/events/create_event_screen.dart`, ARB files

- Убран fallback на `_cityId` при отображении города в CreateEventScreen. Вместо `_cityName ?? _cityId ?? l10n.profileNotSpecified` — `_cityName ?? l10n.eventCreateSelectCity`.
- Добавлен `debugPrint` в catch блок загрузки города.
- i18n: ключ `eventCreateSelectCity` = «Выберите город» / «Select city».

### 1.2. Основатель клуба в API и UI

**Файлы:** `backend/src/api/clubs.routes.ts`, `mobile/lib/shared/models/club_model.dart`, `mobile/lib/features/club/club_details_screen.dart`

**Backend:**
- GET /api/clubs/:id теперь возвращает `creatorId` и `creatorName`.
- `creatorName` формируется из `firstName + lastName` через `usersRepo.findById(club.creatorId)`.

**Mobile:**
- `ClubModel` расширена полями `creatorId: String?` и `creatorName: String?` + парсинг в `fromJson`.
- На `ClubDetailsScreen` после описания добавлен `ListTile` с иконкой `Icons.star` и именем основателя (при наличии `creatorName`).

**i18n:** ключ `clubFounder` = «Основатель» / «Founder».

### 1.3. Экран списка клубов по городу

**Файлы:** `mobile/lib/features/club/clubs_list_screen.dart` (новый), `mobile/lib/app.dart`, `mobile/lib/features/profile/my_clubs_screen.dart`

- Создан `ClubsListScreen` — StatefulWidget, принимает `cityId`, загружает клубы через `ClubsService.getClubs(cityId:)`.
- FutureBuilder → ListView с карточками клубов (название, город, участники).
- GoRouter: добавлен route `/clubs` с query param `cityId`.
- На `MyClubsScreen` добавлена кнопка «Все клубы города» в AppBar.

**i18n:** `clubsListTitle`, `clubsListEmpty`, `clubsListAllClubs`.

## Фаза 3 — Large tasks

### 3.1. Выбор точки старта на карте

**Файлы:** `mobile/lib/features/map/location_picker_screen.dart` (новый), `mobile/lib/app.dart`, `mobile/lib/features/events/create_event_screen.dart`

- Создан `LocationPickerScreen` — полноэкранная YandexMap с пином по центру (center-pin подход, не PlacemarkMapObject).
- Принимает `initialLatitude`/`initialLongitude`.
- По тапу на карту: перемещение камеры + обновление координат.
- Кнопка «Подтвердить» → `Navigator.pop(context, {'lat': ..., 'lon': ...})`.
- GoRouter: route `/map/pick` с query params `lat`/`lon`.
- В `CreateEventScreen`: убраны TextFormField для lat/lon, заменены на кнопку «Выбрать на карте» и state `_selectedLat`/`_selectedLon`.

**i18n:** `eventCreatePickLocation`, `eventCreateLocationSelected`, `eventCreateLocationRequired`, `locationPickerTitle`, `locationPickerConfirm`.

### 3.2. Выход лидера из клуба

**Файлы:** `backend/src/api/clubs.routes.ts`, `backend/src/db/repositories/club_members.repository.ts`, `mobile/lib/features/club/club_details_screen.dart`, `mobile/lib/features/club/transfer_leadership_screen.dart` (новый), `mobile/lib/shared/api/clubs_service.dart`, `mobile/lib/shared/api/api_client.dart`, `mobile/lib/app.dart`

**Backend:**
- POST /api/clubs/:id/leave: добавлена валидация роли leader — если >1 участников, возвращается 400 `leader_cannot_leave`.
- Новый DELETE /api/clubs/:id: роспуск клуба (только leader). Деактивирует всех участников (`deactivateAllMembers`), устанавливает статус `disbanded`.
- `ClubMembersRepository`: добавлен метод `deactivateAllMembers(clubId)`.

**Mobile:**
- `_onLeaveClub()` переписан: для лидера с >1 участниками — диалог «Передать лидерство / Распустить / Отмена»; для одинокого лидера — диалог подтверждения роспуска.
- Добавлен метод `_disbandClub()`.
- Создан `TransferLeadershipScreen` — загружает участников, по тапу: подтверждение → PATCH role=leader.
- `ClubsService`: добавлен `disbandClub(clubId)` → DELETE /api/clubs/:id.
- `ApiClient`: добавлен метод `delete()`.
- GoRouter: route `/club/:id/transfer-leadership`.

**i18n:** `leaderCannotLeave`, `transferLeadership`, `disbandClub`, `disbandConfirm`, `selectNewLeader`, `transferSuccess`, `disbandSuccess`.
