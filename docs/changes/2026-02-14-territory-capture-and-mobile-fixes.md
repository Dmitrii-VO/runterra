# Изменения 2026-02-14: Захват территорий и надежность Mobile

## Цель
Реализовать проверку прав на захват территорий (ADR-0007) и повысить надежность мобильного приложения за счет исправления паттернов загрузки данных и работы с WebSocket.

## Изменения

### Backend
- **Захват территорий (ADR-0007):**
  - Новый эндпоинт: `POST /api/territories/:id/capture`.
  - Валидация: добавлена `CaptureTerritorySchema` (проверка `clubId` как UUID).
  - Логика: разрешено вносить вклад в захват только пользователям с **активным** статусом в указанном клубе (`status === 'active'`).
  - Ошибки: возвращается `403 Forbidden` с деталями (`status`, `requiredStatus`), если условие не выполнено. Формат ошибок соответствует ADR-0002.
- **Тестирование:**
  - Создан `backend/src/api/territories.routes.test.ts`.
  - Покрыты сценарии: успех, отказ (pending member), отказ (не член клуба), 401 (нет токена), 404 (нет территории).

### Mobile
- **WebSocket и Polling (Надежность):**
  - `ChatWebSocketService`: добавлен статус соединения `WsStatus` и `statusStream`.
  - Авто-реконнект: сервис теперь сам пытается переподключиться каждые 5 секунд при ошибке.
  - Реактивный UI: `ClubMessagesTab` автоматически включает polling только при отсутствии WS-соединения, слушая `statusStream`.
- **Исправление FutureBuilder (Производительность):**
  - `CityPickerDialog`: переведен на `StatefulWidget` внутри диалога. `Future` списка городов теперь кэшируется в `initState`, предотвращая дубликаты запросов при перерисовке.
  - `MapScreen`: список клубов в BottomSheet вынесен в отдельный виджет `_ClubsBottomSheet` с кэшированием `Future`.
- **Локализация и Стандарты:**
  - Подтверждено использование `AppLocalizations` и `ServiceLocator` во всех затронутых компонентах.

## Инструкции по проверке
1. **Backend:** Запустить `npm test src/api/territories.routes.test.ts` в папке `backend`.
2. **Mobile (WS):** Отключить интернет во время открытого чата — через 5 секунд после восстановления сети чат должен переподключиться автоматически.
3. **Mobile (Future):** Открыть профиль -> Выбор города. Проверить в логах, что запрос списка городов происходит только один раз при открытии диалога.

## Примечания
- ADR-0007 теперь технически соблюдается на уровне API.
- Глобальная очистка `FutureBuilder` повысила стабильность UI на слабых устройствах.
