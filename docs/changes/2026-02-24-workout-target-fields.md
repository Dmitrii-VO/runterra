# Добавление целевых значений в тренировки (2026-02-24)

## Контекст
При создании тренировки тренер мог выбрать только «Целевую метрику» (например, Дистанция или Время), но не мог указать конкретное значение (например, 5 км или 30 минут) и интенсивность (пульсовую зону). Это делало карточку тренировки недостаточно информативной для спортсменов.

## Выполненные изменения

### 1. Бэкенд
- Создана миграция базы данных `028_workouts_target_fields.sql`, добавляющая колонки `target_value` (целое число) и `target_zone` (строка, до 50 символов) в таблицу `workouts`.
- Обновлена модель `Workout` и Zod-схемы `CreateWorkoutSchema` / `UpdateWorkoutSchema` (добавлены опциональные поля `targetValue` и `targetZone`).
- Обновлен `WorkoutsRepository`: новые поля маппятся при чтении (`rowToWorkout`) и сохраняются при операциях `create` и `update`.
- Эндпоинты `POST /api/workouts` и `PATCH /api/workouts/:id` теперь принимают и сохраняют новые параметры.

### 2. Мобильный клиент
- Обновлена модель `Workout` в `shared/models/workout.dart` (добавлены поля `targetValue` и `targetZone`).
- В `WorkoutsService` метод `createWorkout` расширен для передачи новых параметров.
- Обновлен UI формы `WorkoutFormScreen`:
  - Под полем выбора целевой метрики добавлено поле ввода `targetValue` (принимает только числа).
  - Лейбл поля `targetValue` динамически меняется в зависимости от выбранной метрики (например, «Дистанция (метры)», «Длительность (минуты)», «Темп (сек/км)»).
  - Добавлен выпадающий список `targetZone` с вариантами пульсовых зон от Z1 до Z5 (и опцией «Не указана»).
- В файлы локализации `app_ru.arb` и `app_en.arb` добавлены переводы для новых полей и опций зон (например, `workoutTargetValueDistance`, `zoneZ1`, `zoneNone`).
