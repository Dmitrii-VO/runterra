# Требование: Меркатели (Mercenaries)

## Контекст

По аудиту модуля клубов (docs/tasks/gemini-clubs-audit-report.md) и product_spec, меркатели — пользователи без клуба, которые могут помогать в захвате территорий.

## Требование к будущей реализации

При реализации логики захвата территорий (cron, обработка runs по product_spec §16):

- **Пользователи без `club_id`** (меркатели) могут выполнять пробежки.
- Пробежки меркателей засчитываются в «активность зоны», делая её **оспариваемой (contested)**.
- Оспариваемая зона упрощает последующий захват любым клубом (облегчённые условия по product_spec).

## Примечание (MVP-решение 2026-02-17)

В ближайшем MVP по реальному захвату территорий принято более жёсткое правило:

- **Пробежки пользователей без клуба не влияют на территории** (не начисляют очки и не делают зону contested).

Вариант “меркатели влияют на contested” остаётся как future-опция и может быть возвращён после появления базового реального подсчёта.

## Текущее состояние

- Таблица `territories` в БД отсутствует (территории — статический конфиг).
- Логика захвата (cron, проверка runs) не реализована.
- Код не изменяется до появления реальной логики захвата.
