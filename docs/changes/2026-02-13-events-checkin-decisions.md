# Решения по check-in событий (Z8, 2026-02-13)

Принятые продуктовые решения для реализации Swipe-to-run и связанных изменений в рамках задачи Z8 «События — выкинуть/перепридумать кнопку „Отметиться“».

## Сводка решений

| # | Решение | Выбор |
|---|---------|-------|
| 1 | UX check-in | **Swipe-to-run** |
| 2 | Связь check-in и Run | **Check-in + Run** (свайп сразу запускает пробежку) |
| 3 | Окно времени | **30 мин до — 1 ч после** старта события |
| 4 | Минимум 1 км в территории | **Проверять** для зачёта check-in |
| 5 | Радиус геозоны | **500 м** (без изменений) |
| 6 | Где показывать | **EventDetailsScreen + фильтр «Участвую»** |
| 7 | Вне условий | **Показывать disabled с пояснением** |

## Детали

### 1. Swipe-to-run
Карточка «Начать пробежку» со свайпом на экране деталей события. Рекомендация brainstorm Gemini (2026-02-13).

### 2. Check-in + Run
Один свайп выполняет:
1. Check-in на событие (POST /api/events/:id/check-in)
2. Старт пробежки (переход на Run flow с контекстом события)

Соответствует product_spec 16.3 (кнопки «Начать пробежку» и «Завершить»).

### 3. Окно времени
- **Было (backend):** 15 мин до — 30 мин после старта
- **Стало:** 30 мин до — 1 ч после старта

Требует изменения в `EventsRepository.checkIn()` и связанных тестах.

### 4. Минимум 1 км в территории
product_spec 5.5: «для зачёта — минимум 1 км в территории». Backend пока не проверяет. Нужно добавить проверку: check-in засчитывается только если пользователь пробежал ≥ 1 км в территории события (или связанной с событием).

**Открытый вопрос:** момент проверки — при завершении пробежки или при check-in? Логично: check-in при старте, проверка 1 км — при завершении пробежки (или при финальном подтверждении).

### 5. Радиус геозоны
Оставляем 500 м. product_spec 200–500 м — для MVP одного значения достаточно.

### 6. EventDetailsScreen + фильтр «Участвую»
- Карточка Swipe-to-run показывается на `EventDetailsScreen`
- Добавить фильтр «Участвую» в `EventsScreen` — показывает только события, где пользователь участник
- Backend: параметр `participantOnly` (или `joinedByMe`) в `GET /api/events`

### 7. Вне условий — disabled с пояснением
Если пользователь не в геозоне или вне окна времени:
- Карточка видна, но неактивна (disabled)
- Текст пояснения: «Подойдите ближе к месту старта» / «Событие ещё не началось» / «Окно check-in закрыто» и т.п.
