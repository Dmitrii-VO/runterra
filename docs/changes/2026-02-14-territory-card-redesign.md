**Дата:** 15.02.2026 (обновлено 16.02.2026)
**Статус:** Реализовано (мок-данные, без DB/GPS/cron — секция 6.5 отложена)
**Цель:** Адаптировать интерфейс карточки территории под механику «Беговых Лиг» — захват через накопление километража с динамическими тирами сложности.

## 1. Механика: Беговые Лиги (League & Conquest)

### 1.1. Концепция

Город (СПб, 18 административных районов) разделён на зоны. Каждая зона имеет **тир сложности**, который определяется динамически на основе беговой активности в районе. Клубы соревнуются за захват зон, набирая километраж за месяц.

### 1.2. Тиры (динамические, пересчёт раз в месяц)

Тир определяется по **совокупной активности всех бегунов** в районе за предыдущий месяц. Метрики:
- **Суммарный объём (км)** — сумма всех километров, пробеганных внутри полигона территории
- **Средний темп (мин/км)** — средний темп всех засчитанных пробежек в районе

#### Формула тира

Каждой метрике присваивается балл от 0 до 3, итоговый тир = max(балл_объёма, балл_темпа):

| Балл | Суммарный объём за месяц | Средний темп     |
|------|--------------------------|------------------|
| 0    | < 50 км                  | > 7:00 мин/км    |
| 1    | 50–200 км                | 5:30–7:00 мин/км |
| 2    | 200–500 км               | 4:30–5:30 мин/км |
| 3    | > 500 км                 | < 4:30 мин/км    |

| Итоговый балл | Тир    | Название         | Цвет хедера           |
|---------------|--------|------------------|-----------------------|
| 0             | Green  | Новички          | Зелёный градиент      |
| 1             | Blue   | Продвинутые      | Синий градиент        |
| 2             | Red    | Специалисты      | Красный градиент      |
| 3             | Black  | Элита            | Тёмный/чёрный градиент|

**Пересчёт:** 1-го числа каждого месяца, одновременно со сбросом очков.

### 1.3. Формула очков

#### Базовые очки
- **1 км внутри полигона территории = 1 балл** (для клуба бегуна)
- Учитывается только GPS-трек, попадающий внутрь полигона (intersection)
- **Минимальная дистанция:** 500 м внутри полигона для засчитывания пробежки (отсечка от случайного пробегания мимо)

#### Бонус за зону (Zone Bounty)
- Бег внутри конкретного района даёт **x1.5** к набранным очкам
- Стимулирует «защиту» своей территории и целенаправленные забеги в конкретные районы

#### Бонус за темп (Pace Multiplier) — мягкий
Пробежка **всегда засчитывается** вне зависимости от темпа. Но если средний темп пробежки лучше порога зоны, применяется дополнительный мультипликатор:

| Тир   | Порог темпа для бонуса | Мультипликатор |
|-------|------------------------|----------------|
| Green | < 7:00 мин/км          | x1.2           |
| Blue  | < 5:30 мин/км          | x1.3           |
| Red   | < 4:30 мин/км          | x1.4           |
| Black | < 4:00 мин/км          | x1.5           |

#### Итоговая формула
```
points = distance_km × zone_bounty(1.5) × pace_multiplier(1.0–1.5)
```

**Пример:** 5 км в Blue-зоне с темпом 5:15 → 5 × 1.5 × 1.3 = **9.75 баллов** клубу.
**Пример:** 5 км в Blue-зоне с темпом 6:00 (не прошёл порог) → 5 × 1.5 × 1.0 = **7.5 баллов** клубу.

### 1.4. Месячный цикл (сезон)

- **Начало сезона:** 1-е число каждого месяца, 00:00 UTC+3
- **Сброс:** все очки клубов по всем территориям обнуляются → status = FREE
- **Пересчёт тиров:** на основе данных прошедшего месяца (см. 1.2)
- **Архив:** данные за прошлый месяц сохраняются в историю (winners, total km, tier)
- **Захват:** клуб с наибольшим количеством очков в зоне на конец месяца — владелец (status = CAPTURED)
- Промежуточный лидер отображается как «текущий владелец», но это не финальный захват

### 1.5. Edge cases

| Ситуация | Поведение |
|----------|-----------|
| Район с 0 активности за месяц | Тир = Green (по умолчанию) |
| Первый месяц работы (нет истории) | Все районы = Green |
| Один клуб в зоне | Автозахват при наборе минимум 10 км (порог защиты от случайных пробежек) |
| Пробежка пересекает 2+ зон | Километраж делится пропорционально по полигонам; бонусы считаются отдельно для каждой зоны |
| Бегун без клуба | Пробежка записывается в личную статистику, но не даёт очков ни одному клубу |
| Ничья (равные очки) | Клуб, набравший очки раньше (по времени первой пробежки в зоне), побеждает |
| Клуб расформирован в середине месяца | Очки клуба сохраняются до конца месяца (другие клубы могут обойти) |

## 2. Концепция UI: «League Tactics»

Совмещаем стиль тактического интерфейса с соревновательной спортивной эстетикой.

- **Акцент:** чёткая индикация тира (цвет), выгода (бонусы), состояние битвы (лидерборд)
- **Эмоция:** «Это моя лига, я могу здесь победить» или «Тут высокий бонус, стоит сбегать»

### 2.1. Карта — отображение тиров

- **Цвет полигона территории** на карте = цвет тира (green / blue / red / black)
- Используются существующие цвета из `territories.config.ts`, но с заменой на тир-цвета
- Прозрачность полигона: 30% fill, 80% stroke
- **Иконка захвата:** если территория CAPTURED — маленький логотип/инициалы клуба-владельца в центре полигона

## 3. Детализация Bottom Sheet

### 3.1. Хедер: Тир зоны и Статус (Zone Header)

Фон хедера — градиент цвета тира для мгновенного считывания сложности.

**Элементы:**
- **Бейдж тира:** иконка + текст, например: `🟢 GREEN ZONE • НОВИЧКИ` или `⚫ BLACK ZONE • ЭЛИТА`
- **Статус захвата:**
  - Захвачена → логотип клуба + `CONTROLLED BY [ClubName]`
  - Свободна → `OPEN SEASON`
  - Оспаривается (2+ клуба с очками) → `CONTESTED` + кол-во клубов

### 3.2. Блок «Условия и Награда» (Rules Grid)

Три карточки в ряд:

| # | Параметр | Иконка | Пример текста |
|---|----------|--------|---------------|
| 1 | **Темп для бонуса** | ⏱ Секундомер | «< 5:30 → x1.3» |
| 2 | **Бонус зоны** | ⭐ Звезда | «x1.5 очков» |
| 3 | **До сброса** | ⏳ Песочные часы | «Сброс через 12д» |

### 3.3. Блок «Битва за район» (Battle Progress)

Динамика борьбы за территорию:

- **Лидер:** аватар/лого клуба + название + «1240 км»
- **Ваш прогресс (если не лидер):** «Ваш клуб: 800 км (3-е место)»
- **Отрыв:** прогресс-бар + текст «−440 км до лидера»
- Если ваш клуб лидирует: «Ваш клуб лидирует! +150 км отрыв»

### 3.4. Панель действий (Action Bar)

- **Primary Button:** Large, full width
  - Текст: **«БЕЖАТЬ ЗА ЗОНУ (+1.5x)»**
  - Подтекст: «В зачет: Blue Zone»
  - Действие: переход на вкладку Run с предвыбранной целевой зоной
- **Secondary:** иконка-кнопка «Таблица лидеров» → открывает лидерборд

### 3.5. Состояния Bottom Sheet

#### Обычное состояние (пользователь в клубе, есть данные)
Полный интерфейс, описанный выше.

#### Пользователь без клуба
- Хедер с тиром — **отображается** (информация о зоне доступна всем)
- Rules Grid — **отображается**
- Battle Progress — **скрыт**, вместо него: блок-заглушка с текстом и CTA
  - Текст: «Вступи в клуб, чтобы соревноваться за территории»
  - Кнопка: **«НАЙТИ КЛУБ»** → переход на `/clubs?cityId=spb`
- Action Bar: Primary скрыт, вместо него — та же кнопка «НАЙТИ КЛУБ»

#### Нет данных за текущий месяц (начало сезона)
- Хедер с тиром (тир рассчитан по прошлому месяцу)
- Rules Grid — отображается
- Battle Progress: текст «Новый сезон начался, данных пока нет. Стань первым!»
- Action Bar — обычный

#### Loading
- Хедер: название территории + skeleton-shimmer вместо бейджа тира
- Rules Grid: три skeleton-карточки
- Battle Progress: skeleton-блок
- Action Bar: кнопка disabled

#### Error
- Хедер: название территории (из кэша карты)
- Тело: иконка ошибки + текст «Не удалось загрузить данные» + кнопка **«Повторить»**

## 4. Лидерборд (Leaderboard)

Отдельный экран/модальный sheet, открывается по кнопке «Таблица лидеров».

### Содержимое:
- **Заголовок:** «[Название района] — Лидерборд»
- **Тир-бейдж** в заголовке (цветовая метка)
- **Список:** топ-10 клубов, отсортированных по очкам (км)
  - Каждая строка: позиция (#1, #2...), лого/инициалы клуба, название, набранные км
  - Клуб пользователя подсвечен (highlight background)
- **Если клуб пользователя не в топ-10:** отдельная строка внизу — «...», «#15 Ваш клуб — 320 км»
- **Пользователь без клуба:** таблица отображается, но без подсветки «вашего» клуба

## 5. UX-сценарии

### Сценарий 1: Опытный бегун в клубе

1. Бегун (уровень Advanced) открывает карту
2. Видит районы, раскрашенные по тирам — его район Blue
3. Тапает на Blue-зону, открывается bottom sheet
4. Видит **синюю шапку**: «BLUE ZONE • ПРОДВИНУТЫЕ»
5. Rules Grid: «Темп < 5:30 → x1.3», «x1.5 очков», «Сброс через 12д»
6. Battle Progress: вражеский клуб «RunnersPro» лидирует с отрывом 15 км
7. **Решение:** жмёт «БЕЖАТЬ ЗА ЗОНУ» → переход на вкладку Run

### Сценарий 2: Новичок без клуба

1. Новый пользователь открывает карту, видит цветные районы
2. Тапает на Green-зону из любопытства
3. Видит **зелёную шапку**: «GREEN ZONE • НОВИЧКИ»
4. Rules Grid: «Темп < 7:00 → x1.2», «x1.5 очков», «Сброс через 12д»
5. Вместо Battle Progress: «Вступи в клуб, чтобы соревноваться за территории»
6. Жмёт **«НАЙТИ КЛУБ»** → экран списка клубов

### Сценарий 3: Начало нового сезона

1. 1-е число месяца, бегун открывает карту
2. Цвета районов обновились (некоторые повысили/понизили тир)
3. Тапает на свой район: «OPEN SEASON», Battle Progress: «Новый сезон начался»
4. Жмёт «БЕЖАТЬ ЗА ЗОНУ» — хочет быть первым в этом месяце

## 6. Техническая реализация

### 6.1. API — расширение TerritoryViewDto

К текущим полям (`id`, `name`, `status`, `coordinates`, `geometry`, `cityId`, `capturedByUserId`, `clubId`, `color`, `createdAt`, `updatedAt`) добавляются:

```typescript
// Новые поля в TerritoryViewDto
{
  // Tier info
  tier: 'green' | 'blue' | 'red' | 'black';
  paceThreshold: string;       // e.g. "5:30" — порог темпа для бонуса
  pointMultiplier: number;     // e.g. 1.3 — мультипликатор за темп
  zoneBounty: number;          // e.g. 1.5 — бонус за бег в зоне (пока всегда 1.5)

  // Season info
  seasonEndsAt: string;        // ISO 8601 date — конец текущего сезона (1-е число следующего месяца)

  // Leaderboard (top-N clubs)
  leaderboard: Array<{
    clubId: string;
    clubName: string;
    totalKm: number;           // Суммарные очки (км × мультипликаторы)
    position: number;          // 1-based
  }>;

  // Progress of the requesting user's club (null if user has no club)
  myClubProgress: {
    clubId: string;
    clubName: string;
    totalKm: number;
    position: number;
    gapToLeader: number;       // km gap to first place (0 if leader)
  } | null;
}
```

**Примечание:** поле `color` переиспользуется: значение определяется тиром (green → `#4CAF50`, blue → `#2196F3`, red → `#F44336`, black → `#212121`).

### 6.4. Модели (Flutter)

На мобайле две модели территории:
- **`TerritoryModel`** (`shared/models/territory_model.dart`) — базовая, без `color` и `geometry`, используется для REST-списков.
- **`TerritoryMapModel`** (`shared/models/territory_map_model.dart`) — расширенная, с `color` и `geometry`, используется для отрисовки на карте и в bottom sheet.

Новые поля лиги (`tier`, `leaderboard`, `seasonEndsAt` и т.д.) добавляются в **`TerritoryMapModel`**, т.к. именно она используется при отображении карточки территории. `TerritoryLeagueInfo` — вложенный объект внутри `TerritoryMapModel`, загружаемый лениво (lazy load) при открытии bottom sheet через отдельный запрос `GET /api/territories/:id`.

```dart
enum ZoneTier { green, blue, red, black }

class TerritoryLeagueInfo {
  final ZoneTier tier;
  final String paceThreshold;     // "5:30"
  final double pointMultiplier;   // 1.3
  final double zoneBounty;        // 1.5
  final DateTime seasonEndsAt;
  final List<LeaderboardEntry> leaderboard;
  final ClubProgress? myClubProgress;
}

class LeaderboardEntry {
  final String clubId;
  final String clubName;
  final double totalKm;
  final int position;
}

class ClubProgress {
  final String clubId;
  final String clubName;
  final double totalKm;
  final int position;
  final double gapToLeader;
}
```

## 7. Совместимость и ограничения

- **18 районов СПб:** конфигурация уже загружена (territories.config.ts), полигоны реальные
