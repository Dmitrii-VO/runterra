# Редизайн карточки территории (Territory Bottom Sheet)

**Дата:** 14.02.2026
**Статус:** Черновик / Предложение
**Цель:** Заменить утилитарное информационное окно на вовлекающий интерфейс, мотивирующий игрока начать пробежку и захват.

## 1. Контекст
В текущей реализации (`TerritoryBottomSheet`) окно выглядит как справочная карточка: название, статус текстом, маленькая кнопка. Это не вызывает эмоций и не объясняет ценность территории для игрока.
**Задача:** Превратить просмотр территории в "получение миссии".

## 2. Концепции UI/UX

В ходе брейншторма были выделены три направления:

### Концепт А: «Tactical Ops» (Тактическая сводка) — **Рекомендованный**
*   **Метафора:** Военная сводка или брифинг перед спецоперацией (как в CoD: Warzone, Ingress).
*   **Акцент:** Противостояние «Свой / Чужой». Четкое цветовое кодирование статусов.
*   **Эмоция:** Азарт, вызов, мобилизация. «Это вражеская точка, её нужно отбить».
*   **Ключевые элементы:** Агрессивные цвета, крупные индикаторы прогресса/статуса, техничный шрифт.

### Концепт Б: «King of the Hill» (Царь горы)
*   **Метафора:** Профиль чемпиона.
*   **Акцент:** Социальное доказательство и тщеславие. Крупный аватар владельца.
*   **Эмоция:** Соперничество. «Я быстрее этого парня».
*   **Ключевые элементы:** Фото владельца/лого клуба на пол-экрана, кнопка «Бросить вызов».

### Концепт В: «Explorer RPG» (Искатель сокровищ)
*   **Метафора:** Карточка найденного артефакта или локации в RPG.
*   **Акцент:** Награда и ресурсы.
*   **Эмоция:** Любопытство, жадность коллекционера.
*   **Ключевые элементы:** Иконки кристаллов/XP, сундуки, мягкая графика.

## 3. Детализация решения («Tactical Ops»)

Выбран стиль **«Tactical Ops»** как наиболее подходящий для механики территориального контроля.

### Структура интерфейса (Сверху вниз)

#### 1. Хедер: Статус и Угроза (Visual Header)
Верхняя часть карточки (фон) должна мгновенно считываться по цвету.
*   **Цветовое кодирование:**
    *   🔴 **Captured (Враг):** Насыщенный красный/оранжевый градиент. Текст: «CONTROLLED BY ENEMY».
    *   🔵 **Captured (Свой/Союзник):** Синий/Зеленый. Текст: «SECURED».
    *   ⚪ **Free (Свободно):** Серый/Нейтральный паттерн. Текст: «NO MAN'S LAND» / «FREE ZONE».
    *   🟡 **Contested (Конфликт):** Желтый с анимацией (пульсация). Текст: «UNDER ATTACK».
*   **Иконка:** Крупный символ статуса (Щит, Флаг, Замок) в левом углу хедера.

#### 2. Блок «Разведданные» (Stats Grid)
Сразу под хедером — ряд из 3-х ключевых показателей (Grid Row). Даже если данных нет на бэкенде, мы вычисляем их на клиенте для геймификации.

1.  **Сложность (Difficulty):**
    *   *Визуал:* 1-5 иконок молнии (⚡).
    *   *Логика:* Рассчитывается на клиенте исходя из периметра/площади полигона. Больше территория = больше молний.
2.  **Площадь (Area):**
    *   *Визуал:* Иконка линейки/карты.
    *   *Текст:* Например, «1.2 км²» или «Периметр: 800м».
3.  **Награда (Bounty) — *Mock Data*:**
    *   *Визуал:* Иконка звезды или монеты.
    *   *Текст:* «+50 XP» (пока захардкодить или привязать к сложности). Это критически важно для мотивации.

#### 3. Блок Владения (Ownership Info)
Если территория захвачена:
*   Горизонтальная плашка.
*   **Слева:** Логотип клуба или Аватар игрока (Circular).
*   **Центр:** Название («Red Runners Club»).
*   **Справа:** Время удержания (например, «3д 12ч» — *можно пока замокать или скрыть*).
*   *Идея:* Если это враг, добавить красную рамку вокруг аватара.

#### 4. Панель Действий (Action Bar)
Закреплена внизу или просто крупный блок в конце.
*   **Primary Button (Основная):**
    *   Огромная кнопка во всю ширину (кроме места под навигацию).
    *   Текст: **«НАЧАТЬ ЗАХВАТ»** (вместо скучного «Подробнее»).
    *   Иконка: Бегущий человек.
    *   Цвет: Контрастный к фону (яркий акцент).
*   **Secondary Button (Вторичная):**
    *   Кнопка «Маршрут» (иконка навигации).
    *   Стиль: Outlined или Icon Button рядом с основной.

## 4. Техническая реализация (Flutter)

### Виджеты
*   `showModalBottomSheet`: Использовать `isScrollControlled: true` для гибкой высоты и `backgroundColor: Colors.transparent` для кастомного скругления верха.
*   **Декор:** `BoxDecoration` с `LinearGradient` для хедера.
*   **Сетка статов:** `Row` + `Expanded` ячейки. Внутри `Column` (Иконка -> Значение -> Подпись).
*   **Асинхронность:**
    *   Данные территории (`TerritoryMapModel`) приходят синхронно из тапа.
    *   Название клуба (`clubId`) нужно грузить асинхронно.
    *   *Решение:* Показывать скелетон (Shimmer) на месте имени клуба, пока идет `clubsService.getClub(id)`.

### Геймификация (Client-side calculation)
Пока на бэкенде нет полей `difficulty` или `reward`, реализуем простую логику на клиенте:
```dart
// Пример логики
int calculateDifficulty(List<Point> points) {
  // Если точек нет (круг) -> 1
  // Если периметр > 1км -> 3
  // Иначе -> 2
  return 2; 
}

int calculateXP(int difficulty) => difficulty * 50;
```

## 5. Итоговый UX сценарий
1.  Игрок видит серую зону на карте.
2.  Тапает по ней.
3.  Всплывает окно с серым хедером "СВОБОДНАЯ ЗОНА".
4.  Видит статы: "Сложность: ⚡⚡ (Легко)", "Награда: +100 XP".
5.  Видит огромную кнопку "НАЧАТЬ ЗАХВАТ".
6.  **Результат:** Игрок понимает, что задача посильная, награда есть, и нажимает кнопку старта.
