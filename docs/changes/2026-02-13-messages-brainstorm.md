# Brainstorm: вкладка Сообщения MVP (2026-02-13)

Результаты brainstorm по задаче «как довести вкладку Сообщения до MVP» (контекст: product_spec, feedback, текущая реализация).

## Контекст

- **Product spec 5.3:** Чат клуба (текстовый, базовый); комментарии к тренировкам.
- **Текущее состояние:**
  - **Клуб:** реализован — список клубов → чат, WebSocket real-time, подгрузка истории, composer.
  - **Личные:** заглушка (пустой экран).
  - **Тренер:** заглушка.
- **Backend:** REST + WebSocket для `club:{clubId}` и `club:{clubId}:{channelId}`; таблица `club_channels`, эндпоинты GET/POST `/api/clubs/:id/channels`.
- **Фидбек (2026-02-12):**
  - Стабилизировать поле ввода после отправки (не «прыгать», клавиатура не закрывать).
  - Несколько подчатов у клуба (тренировки, болталка).

## MVP scope: что обязательно

### 1. Клубный чат — довести до стабильного MVP

| Элемент | Статус | Действие |
|---------|--------|----------|
| Список клубов → чат | ✅ | — |
| WebSocket real-time | ✅ | — |
| Отправка/получение сообщений | ✅ | — |
| Поле ввода после отправки | ⚠️ | **Z7:** стабилизировать (FocusNode, не unfocus, guard scroll) |
| Подчаты (каналы) | Backend ✅, Mobile частично | **Опционально для MVP:** один канал «Общий» достаточен; подчаты — post-MVP |

**Вывод:** Минимум для MVP клубного чата — **Z7 (стабилизация ввода)**. Подчаты можно отложить.

## Варианты по табам «Личные» и «Тренер»

| Вариант | Описание | Плюсы | Минусы |
|---------|----------|-------|--------|
| **A. Оставить заглушки** | Личные и Тренер — пустые экраны с текстом «Скоро» | Минимум работы, фокус на клубе | Пользователь видит нерабочие табы |
| **B. Скрыть табы** | Показывать только таб «Клуб» | Чистый UX, нет мёртвых зон | Меняется навигация, нужна переработка MessagesScreen |
| **C. Личные в MVP** | Реализовать 1:1 чат (backend + mobile) | Полноценные сообщения | Большой объём: новая модель, API, UI, WebSocket |
| **D. Тренер в MVP** | Чат тренер ↔ участники | Соответствует product spec | Роль тренера не проработана (Z9), scope размыт |

### Рекомендация для MVP

**Вариант A или B.**

- **A** — если хочется сохранить структуру (3 таба) и не трогать навигацию. Заглушки с понятным текстом («Личные чаты — в разработке», «Сообщения тренера — в разработке»).
- **B** — если приоритет «только рабочий функционал»: убрать табы Личные/Тренер, оставить один экран «Чат клуба» (список клубов → чат). Меньше путаницы для пользователя.

**Личные и Тренер — отложить** до post-MVP: после стабилизации клубного чата и проработки роли тренера (Z9).

**Принято (2026-02-13):** Вариант A — оставить 3 таба с заглушками. Тексты обновлены: «Личные чаты — в разработке», «Сообщения тренера — в разработке». Реализовано в docs/changes/2026-02-13-messages-mvp-stubs.md.

## Комментарии к тренировкам (product spec)

| Вариант | Описание | Сложность |
|---------|----------|-----------|
| **Встроить в чат события** | Отдельный подчат «Тренировки» с привязкой к eventId | Средняя (Z5/Z6) |
| **Отдельная секция на экране события** | Комментарии под карточкой события (EventDetailsScreen) | Низкая |
| **Отложить** | Не в MVP | — |

**Рекомендация:** Для MVP **отложить**. Комментарии к тренировкам логичнее делать на экране события (EventDetailsScreen), а не в общем чате. Это отдельная фича, не блокирует MVP вкладки Сообщения.
