## 1. Профиль — перенос настроек в «Редактирование профиля»

### Запрос

Перенести из раздела **Профиль** в раздел **Редактирование профиля** следующие элементы:

- **Геолокация** — тап открывает настройки приложения
- **Видимость профиля** — переключатель
- **Выйти из аккаунта**
- **Удалить аккаунт**

### Текущее состояние

- Эти элементы находятся в `ProfileSettingsSection` на основном экране профиля (`ProfileScreen`).
- Экран «Редактирование профиля» (`EditProfileScreen`) содержит форму: имя, фамилия, страна, дата рождения, пол, город, URL фото.

### Задачи для реализации

1. Убрать `ProfileSettingsSection` с `ProfileScreen` (или оставить только не-настройки, если есть).
2. Добавить в `EditProfileScreen` блок с:
   - Геолокация (ListTile → `Geolocator.openAppSettings()`)
   - Видимость профиля (Switch + `UsersService.updateProfile(profileVisible:)`)
   - Выйти из аккаунта (ListTile → диалог подтверждения → `AuthService.signOut()`)
   - Удалить аккаунт (ListTile → диалог подтверждения → `UsersService.deleteAccount()`)
3. `EditProfileScreen` должен получать `ProfileModel` или `ProfileUserData` + `profileVisible` для отображения Switch.
4. При выходе/удалении — `context.go('/login')` после signOut.

### Файлы

- `mobile/lib/features/profile/profile_screen.dart` — убрать `ProfileSettingsSection`
- `mobile/lib/features/profile/edit_profile_screen.dart` — добавить блок настроек
- `mobile/lib/shared/ui/profile/settings_section.dart` — возможно переиспользовать или вынести логику в EditProfileScreen

## 2. Вкладка События — перепроверка фильтров

### Контекст

Вкладка События недавно дорабатывалась (фильтры, swipe-to-run). Текущий набор фильтров зафиксирован в docs/changes/events.md (раздел «Текущее состояние фильтров»). Требуется перепроверка работы: **Сегодня**, **Завтра**, **7 дней**, **Только открытые**.

### Результат проверки

| Фильтр           | Mobile (events_screen.dart) | Backend (events.repository.ts) | Статус |
|------------------|-----------------------------|--------------------------------|--------|
| Сегодня          | `dateFilter: 'today'`      | `dateFilter === 'today'`       | ✅     |
| Завтра           | `dateFilter: 'tomorrow'`    | `dateFilter === 'tomorrow'`    | ✅     |
| 7 дней           | `dateFilter: 'next7days'`   | `dateFilter === 'next7days'`   | ✅     |
| Только открытые  | `onlyOpen: _onlyOpen`       | `onlyOpen` передаётся и применяется | ✅     |

### Детали

- **Фильтры по дате:** Mobile передаёт `dateFilter` в `EventsService.getEvents()`, backend принимает и применяет в `findAll()`. Диапазоны: today = сегодня 00:00–завтра 00:00; tomorrow = завтра 00:00–послезавтра 00:00; next7days = сегодня 00:00–через 7 дней.
- **Только открытые:** Backend (2026-02-14) поддерживает `onlyOpen=true`: при `onlyOpen === true` в `EventsRepository.findAll()` применяется условие `status = 'open'`. Mobile передаёт `onlyOpen` в query; дополнительно клиентская фильтрация (events_screen.dart) остаётся как safety net.

### Рекомендации

1. **Ручная проверка:** Пройти сценарии: выбрать «Сегодня» / «Завтра» / «7 дней» — список должен обновляться; включить «Только открытые» — не должны показываться полные и прошедшие события.
2. **Реализовано (backend):** Добавлен параметр `onlyOpen=true` в `GET /api/events`; при `onlyOpen === true` в `EventsRepository.findAll()` применяется условие `status = 'open'`.
