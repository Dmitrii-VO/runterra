# Рейтинг клубов в городе (Лидерборд) (2026-02-24)

## Контекст
Был запрос на реализацию осмысленного содержимого для Bottom Sheet «Рейтинг в городе» на экране деталей клуба. В качестве MVP была выбрана простая модель лидерборда:
- Очки клуба = Количество участников * 1 + Количество территорий (где клуб лидирует) * 10.
- Отображение: топ-10 клубов города + собственный клуб пользователя (даже если он не в топ-10), чтобы всегда можно было видеть свое место.

## Выполненные изменения

### 1. Backend
- Создана функция-помощник `getCityLeaderboard` в `src/api/clubs.routes.ts`, которая вычисляет очки для всех клубов города, сортирует их по убыванию очков (а затем по алфавиту) и присваивает ранги.
- Добавлен новый эндпоинт `GET /api/clubs/leaderboard/:cityId?clubId=...` возвращающий JSON с `leaderboard` (до 10 клубов) и `myClub` (объект текущего клуба, если он запрашивался).
- Существующий эндпоинт `GET /api/clubs/:id` обновлен: теперь он использует `getCityLeaderboard` для согласованного вычисления `membersCount`, `territoriesCount` и `cityRank`. Ранее эти значения вычислялись локально (при этом `territoriesCount` всегда был равен 0 из-за отсутствия связей со скорингом).

### 2. Mobile
- Добавлена новая модель `CityLeaderboardEntry` (в `mobile/lib/shared/models/city_leaderboard_entry.dart`).
- В `ClubsService` добавлен метод `getCityLeaderboard`, реализующий вызов нового эндпоинта.
- На экране `ClubDetailsScreen` переписан метод `_showLeaderboardSheet`. Теперь он использует `FutureBuilder`, который вызывает API и рендерит список клубов (с выделением текущего клуба `isMyClub`).
- Добавлены новые ключи в файлы локализации (`app_ru.arb` и `app_en.arb`) для корректного отображения: `clubLeaderboardSubtitle`, `clubLeaderboardPoints` и `clubMetricPlaceholder`.
