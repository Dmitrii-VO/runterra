# Спецификация: Расписание (Версия 4.1 — Полный Цикл)

## 1. Иерархия отображения (Conflict Resolution)
*   **Приоритет:** Если на один и тот же день у пользователя есть **Личное задание** и **Групповая тренировка** клуба:
    *   Личное задание отображается как основное.
    *   Групповая тренировка скрывается (или уходит в раздел "Также в клубе сегодня"), чтобы не создавать каши.

## 2. Связь с GPS-трекингом (План -> Факт)
1.  **Интеграция в таб "Run":** Перед стартом пробежки пользователю предлагается выбрать "Цель" из запланированных на сегодня заданий.
2.  **Авто-отчет:** После завершения пробежки данные (дистанция, время, трек) прикрепляются к записи в календаре.
3.  **Вид для Тренера:** В списке участников тренер видит статус заданий:
    *   ⚪️ План (еще не начато)
    *   🟢 Выполнено (есть привязанная пробежка)
    *   🔴 Пропущено (день прошел, пробежки нет)

## 3. Управление (Trainer Roster)
*   Любой тренер клуба имеет доступ к списку участников.
*   Тренер может "присвоить" участнику Индивидуальный План, что отключит для него общий график клуба.

## 4. Технические изменения
*   **`activities`**: Добавить поле `scheduledItemId` (ссылка на `events` или `personal_notes`).
*   **Unique Constraint**: Жесткая защита от дублей `(templateId, date)` на уровне БД.
*   **Timezone**: Все расчеты по времени города клуба.
*   **Safety Check (Template Deletion):** При удалении элемента из шаблона (Weekly Template), все будущие записи, сгенерированные по этому `template_id` (где `is_manually_edited = false`), помечаются как `deleted_at`. Это предотвращает "фантомные" тренировки и позволяет создавать новые шаблоны без конфликтов.

## 5. План разработки

1.  **Backend (DB):** 
    *   Таблицы `weekly_schedule_items` (клубные) и `personal_schedule_items` (личные).
    *   Поля связи в `events` и `activities`.
    *   Миграция по очистке старых членств (1 пользователь = 1 клуб).
2.  **Backend (API):** 
    *   Эндпоинты управления шаблонами.
    *   Эндпоинт "Roster" (список участников клуба с их статусами).
3.  **Backend (Cron):** Умная генерация +1 день вперед для всех типов планов.
4.  **Mobile:** 
    *   Экран управления расписанием (для тренера).
    *   Календарная лента с фильтрацией Личное/Групповое (для бегуна).
    *   Связка таба "Run" с текущим планом.
5.  **Notifications:** Пуши об изменениях и напоминания о тренировках.
