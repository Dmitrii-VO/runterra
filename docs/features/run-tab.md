# Вкладка «Пробежка»

Функционал вкладки «Пробежка» мобильного приложения Runterra.

## Назначение

GPS-трекинг пробежек, журнал тренировок, просмотр деталей и статистики. Точка входа для записи и анализа беговой активности.

## Роутинг

- **Активная сессия** (idle/running/paused/completed) → `RunTrackingScreen`
- **Нет сессии** → `RunHistoryScreen` (журнал тренировок)

## RunHistoryScreen — журнал тренировок

**Содержимое:**
- Карточка статистики: общее количество пробежек, суммарная дистанция, средний темп
- Список пробежек (пагинация): дата, дистанция, время, темп
- FAB «Начать пробежку» → переход на RunTrackingScreen
- Тап по пробежке → `RunDetailScreen` (`/run/detail/:id`)

**Состояния:** loading, empty (подсказка «Начните первую пробежку»), error + retry

## RunTrackingScreen — трекинг

### Состояния сессии

| Состояние | Описание | Кнопки |
|-----------|----------|--------|
| idle | Готов к старту | «Старт» |
| running | Идёт запись GPS | «Пауза», «Завершить» |
| paused | Таймер и GPS остановлены | «Продолжить», «Завершить» |
| completed | Результат пробежки | «Назад» (в журнал) |

### Во время бега

- **Карта RunRouteMap:** полилиния маршрута, стрелка направления (по `heading` из GPS), камера следует за последней точкой
- **Таймер:** накопленное активное время (паузы не учитываются)
- **Дистанция:** сумма расстояний между GPS-точками
- **GPS-статус:** индикатор качества сигнала

### Экран результата (completed)

Сетка метрик 2×3:
- Длительность (duration)
- Дистанция (км)
- Темп (мин/км)
- Средняя скорость (км/ч)
- Калории (~65 ккал/км, оценка)
- Пульс (placeholder «—», будущая интеграция Health Connect)

После успешной отправки — кнопка «Назад» возвращает в журнал.

## RunDetailScreen — детали пробежки

- Статическая карта с полилинией маршрута, маркерами старт (зелёный) / финиш (красный)
- Автозум на bounding box трека
- Сетка метрик (аналогично экрану результата)

## Swipe-to-run (интеграция с событиями)

При переходе с `EventDetailsScreen` с карточки SwipeToRunCard:
- Выполняется check-in на событие
- Запускается пробежка с привязкой к событию
- Условия: пользователь записан, в геозоне (500 м от старта), в окне (30 мин до — 1 ч после старта)

## API

| Метод | Назначение |
|-------|------------|
| POST /api/runs | Создание пробежки (duration, distance, gpsPoints) |
| GET /api/runs | Список пробежек пользователя (limit, offset) |
| GET /api/runs/stats | Статистика (totalRuns, totalDistance, totalDuration, averagePace) |
| GET /api/runs/:id | Детали пробежки с GPS-треком |

## Навигация

- `/run` — вкладка (роутер журнал/трекинг)
- `/run/detail/:id` — детали пробежки
