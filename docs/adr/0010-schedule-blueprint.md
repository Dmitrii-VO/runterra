# ADR-0010: Подход Blueprint для вкладки «Расписание» в Клубе

## Статус
Принято (2026-03-13)

## Контекст

Вкладка «Расписание» в клубе позволяет лидерам и тренерам создавать недельные шаблоны тренировок (`weekly_schedule_items`). Изначально предполагалось, что `ScheduleGeneratorService.generateNextMonth()` будет автоматически создавать реальные `Events` из этих шаблонов через cron-задачу.

Проблемы с автоматической генерацией:
- `generateNextMonth()` нигде не вызывался — мёртвый код
- Генератор создаёт события с `startLocation: { longitude: 0, latitude: 0 }` (TODO), что делает их невидимыми на карте и непригодными для Check-in
- Логика синхронизации при изменении шаблона (PATCH/DELETE) уже реализована в `syncTemplateChanges()` — но без активной генерации это no-ops
- Автогенерация на 31 день вперёд плодит «мусорные» события без реальной привязки к месту

## Решение

**Blueprint-подход**: шаблон расписания — это «чертёж», а не расписание само по себе. Лидер/тренер создаёт шаблон один раз, а когда нужно провести тренировку — нажимает кнопку «Провести» рядом с шаблоном. Это открывает стандартный экран создания Event с предзаполненными полями (название, тип, время, workoutId). Тренер добавляет только локацию — и Event создаётся в системе, появляется на карте и в ленте тренировок.

## Изменения

- `generateNextMonth()` помечен `@deprecated`, не удаляется (для истории)
- `syncTemplateChanges()` остаётся без изменений (корректно работает при наличии Events, no-ops сейчас)
- Кнопка «Провести» (`Icons.play_circle_outline`) добавлена к каждому workout-шаблону (не к заметкам)
- Навигация через `/event/create?type=&name=&time=&workoutId=` с query params
- `CreateEventScreen` расширен параметрами: `initialName`, `initialTime`, `initialWorkoutId`

## Последствия

**Плюсы:**
- Каждое Event имеет реальную локацию — корректно отображается на карте, доступен Check-in
- Лидер контролирует когда и где проходит тренировка
- Нет «зомби-событий» с нулевыми координатами в БД
- Простая реализация, минимальный impact на существующий код

**Минусы:**
- Создание Event требует ручного действия (нажать «Провести» + указать локацию)
- Нет автоматического расписания для участников на месяц вперёд

**Ограничения:**
- В будущем, если потребуется автогенерация, нужно добавить поле `base_location` в `clubs` или `weekly_schedule_items` как точку сбора по умолчанию, и подключить генератор к cron с горизонтом 7 дней (не 31).
