# ADR-0007: Захват территорий только активными клубами

## Контекст

По product_spec и аудиту модуля клубов (gemini-clubs-audit-report.md) территория захватывается клубом при выполнении условий (N пробежек, M уникальных участников за период T). Клубы имеют статусы: `pending`, `active`, `inactive`, `suspended`.

Клуб в статусе `pending` создаётся с 1 участником (создатель) и переводится в `active` при достижении 2+ активных членов (авто-активация при approve membership).

## Решение

**При реализации логики захвата территорий** (cron job, API или иной механизм проверки условий) необходимо учитывать:

- Захват территории разрешён **только для клубов в статусе `active`**.
- Клубы в статусе `pending`, `inactive`, `suspended` не могут захватывать территории.

Это обеспечивает, что на карте отображаются только «полноценные» клубы с минимум 2 участниками.

## Статус

- Логика захвата территорий ещё не реализована (территории — статический конфиг).
- Правило зафиксировано для применения при реализации cron/обработки runs по product_spec §16.
