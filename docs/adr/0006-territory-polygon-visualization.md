# ADR 0006: Полигоны вместо кругов для визуализации территорий

**Статус:** Принято  
**Дата:** 2026-02-13  
**Контекст:** docs/tasks/gemini-map-zones-analysis-report.md

## Контекст

Территории отображались как круги (CircleMapObject) с фиксированным радиусом 500 м. Круги создают наложения и пустые зоны на границах, что делает логику «захвата территории» неоднозначной.

## Решение

Для MVP использовать **квадратные полигоны** вместо кругов:

1. **API:** TerritoryViewDto получает опциональное поле `geometry?: GeoCoordinates[]` — массив точек полигона.
2. **Backend:** Функция `generateSquareGeometry(lat, lon, sizeInMeters)` генерирует квадрат вокруг центра. Приближение: 1° ≈ 111320 м (для масштаба города искажениями пренебрегаем).
3. **Mobile:** При наличии `geometry` — PolygonMapObject; при отсутствии — CircleMapObject (fallback для обратной совместимости).

## Альтернативы

| Вариант | Сложность | Результат |
|--------|-----------|----------|
| Квадратная сетка (принято) | Низкая | Бесшовное покрытие, простая проверка попадания |
| Гексагональная (H3) | Средняя | Игровой вид, нужны библиотеки |
| Круги (было) | Низкая | Наложения и пустые зоны |

## Последствия

- ✅ Бесшовное покрытие города (при будущей нарезке сеткой)
- ✅ Обратная совместимость: fallback на круги при отсутствии geometry
- ⚠️ Искажения на больших расстояниях — для масштаба города (<50 км) приемлемо
