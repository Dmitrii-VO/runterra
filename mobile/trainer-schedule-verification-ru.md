# Чек-лист: Этап 5 — Интерфейс Тренера (Mobile)

Используйте этот список для проверки корректности реализации функционала управления расписанием и составом клуба.

## 1. Доступ и интеграция (Экран деталей клуба)
- [x] **Кнопки управления:** Для пользователей с ролью «Лидер» или «Тренер» в деталях клуба виден блок «Управление тренера».
- [ ] **Ограничение доступа:** Обычные участники (`member`) не видят кнопок управления расписанием и планами. (Пропущено)
- [x] **Навигация:** Кнопка «Расписание» открывает экран шаблонов, кнопка «Состав и планы» открывает список участников.

## 2. Управление недельным расписанием (Шаблоны)
- [x] **Селектор дней:** В верхней части экрана есть переключатель Пн–Вс (7 дней).
- [ ] **Список заданий:** При смене дня отображаются соответствующие элементы шаблона. (Не удалось проверить, т.к. добавление заданий не работает)
- [ ] **Добавление заметки:** (Ошибка: вместо диалога появляются кнопки "Workout" и "Personal", которые не создают заметку)
    - [ ] Через кнопку `+` можно создать новую заметку (Note).
    - [ ] Указывается время (HH:mm) и текст задания.
    - [ ] Заметка сохраняется и появляется в списке выбранного дня.
- [ ] **Удаление:** Элемент удаляется из списка при нажатии на иконку удаления. (Не удалось проверить, т.к. добавление заданий не работает)
- [x] **Обработка пустого состояния:** Если заданий на день нет, отображается текст «Нет данных».

## 3. Состав клуба (Roster)
- [x] **Список участников:** Отображаются все активные члены клуба.
- [x] **Тип плана:** У каждого участника виден статус плана: **«Клубный»** или **«Личный»**.
- [ ] **Цветовая индикация:** Статусы визуально отличаются (синий для клубного, фиолетовый для личного). (Не проверялось: нет возможности проверить цвет)
- [x] **Переход:** Тап по участнику открывает экран управления его персональным расписанием.

## 4. Индивидуальные планы (Personal Schedule)
- [x] **Просмотр текущего плана:** Экран показывает персональный шаблон участника.
- [x] **Замена плана:** Кнопка «Заменить персональный план» позволяет перезаписать текущий шаблон (в MVP — установка тестового задания).
- [ ] **Обновление статуса:** После назначения личного плана статус участника в общем списке (Roster) меняется автоматически. (Ошибка: статус не обновляется в списке даже после принудительного обновления)

## 5. Техническая реализация и UX
- [ ] **Локализация:** Все тексты соответствуют русскому языку. (Ошибка: большая часть интерфейса на английском)
- [ ] **Загрузка данных:** Присутствуют индикаторы загрузки при обращениях к API. (Не проверялось: нет возможности симулировать)
- [ ] **Обработка ошибок:** При сбоях сети выводятся соответствующие уведомления (SnackBar). (Не проверялось: нет возможности симулировать)
- [ ] **Обновление по требованию:** На экранах расписания и состава работает кнопка «Обновить» в AppBar. (Ошибка: кнопка не обновляет статус плана участника)
